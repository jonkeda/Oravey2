# Spatial Specification Rendering Integration (Phase 4, Step 14)

## Overview

This implementation integrates spatial specification rendering into the Oravey2 game engine, enabling buildings, roads, and water bodies to be rendered from data-driven specifications generated by the map generation pipeline.

## Architecture

### Core Components

#### 1. **SpatialSpecRenderer** (`Oravey2.Core.World.Spatial`)
- Orchestrates rendering of buildings, roads, and water from `TownSpatialSpecification`
- Manages z-ordering: Terrain â†’ Water â†’ Roads â†’ Buildings
- Performs collision detection across all element types
- Provides detailed logging and statistics

#### 2. **GeoToTileTransformer** (`Oravey2.Core.World.Spatial`)
- Transforms geographic coordinates (lat/lon) to game tile coordinates
- Supports:
  - Point transformations (single lat/lon â†’ tile)
  - Line transformations (road segments â†’ tile series)
  - Polygon transformations (water bodies â†’ tile regions)
  - Rectangle transformations (buildings â†’ rotated footprints)
- Uses Bresenham line algorithm for accurate rasterization
- Handles building rotation via matrix transformations

#### 3. **Shared Types** (`Oravey2.Contracts.Spatial`)
Placed in Contracts to avoid circular dependencies:
- `TownSpatialSpecification` - Complete town layout
- `BuildingPlacement` - Individual building with rotation/dimensions
- `RoadNetwork` - Connected road graph
- `SpatialWaterBody` - Polygon-based water regions
- `BoundingBox` - Geographic bounds
- `SpatialWaterType` enum - River/Canal/Harbour/Lake

### Rendering Pipeline

```
1. Terrain (base layer)
   - Unchanged if no terrain description updates needed
   - Provides foundation for all other elements

2. Water Bodies (second layer)
   - Polygon boundaries rasterized to tiles
   - Set water level on each tile
   - Can overlap terrain only

3. Roads (third layer)
   - Road edges transformed to tile paths
   - Sets Asphalt surface type
   - Can overlap terrain/water

4. Buildings (top layer)
   - Rectangular footprints rotated and rasterized
   - Sets StructureId and clears Walkable flag
   - Collision detection with water/roads/buildings
```

### Collision Detection

Detects and logs overlaps between:
- Building â†” Building (`BuildingBuildingOverlap`)
- Building â†” Water (`BuildingWaterOverlap`)
- Water â†” Building (`WaterBuildingOverlap`)
- Road â†” Building (`RoadBuildingOverlap`)
- Road â†” Water (`RoadWaterOverlap`)

Format: `[COLLISION] {message}` with type classification.

## Integration Points

### Usage in RegionLoader

```csharp

// After terrain is built but before entity spawning

// Log collisions for debugging

    logger.LogWarning("Map has {Count} spatial collisions", result.CollisionCount);

// Statistics available

Debug.Assert(result.RoadStats.SegmentCount > 0);
```

### Coordinate System

- **Input**: WGS84 lat/lon (real-world)
- **Output**: Game tile coordinates (0 to map width/height)
- **Conversion**: Maps bounding box to full map grid
- **Precision**: Sub-degree coordinates handled via normalization

## Performance

- **Building rendering**: O(n * m) where n = buildings, m = avg tiles per building
- **Road rendering**: O(e * d) where e = edges, d = avg distance per edge
- **Water rendering**: O(w * p) where w = water bodies, p = avg polygon complexity
- **Collision detection**: Linear pass over affected tiles

Typical town (4 buildings, 2 roads, 1 water body): <1ms render time.
