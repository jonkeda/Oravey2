# Step 4 — AI

**Goal:** Utility-based enemy AI and schedule-based civilian AI with pathfinding.

**Depends on:** Steps 1, 3

---

## Deliverables

1. `AIBehaviorComponent` — behavior type (Combat/Civilian), sensor config, blackboard reference
2. Sensor system: `SightSensor` (cone, configurable angle + range), `HearingSensor` (radius)
3. `AIBlackboard` — per-entity shared memory: last known target pos, threat level, alert state
4. Utility scorer: each action (Patrol, Investigate, Attack, Flee, Take Cover) scored by weighted considerations
5. `AICombatProcessor` — runs utility AI for combat entities each frame when InCombat or Alert
6. AI states: Idle → Patrol → Alert → Engage → Flee — transitions driven by sensor input + utility scores
7. `AICivilianProcessor` — schedule-driven routines: waypoint list + time-of-day triggers
8. Pathfinding integration: A* on tile grid (nav-mesh deferred to Step 7 with world streaming)
9. `AIGroupCoordinator` — simple group tactics: flanking, focus fire, retreat threshold
10. Unit tests for utility scoring, pathfinding, sensor detection

---

## AI Decision Example

```
Attack score  = weapon_available(0.3) × target_in_range(0.4) × health_ok(0.3)
Flee score    = low_health(0.5) × outnumbered(0.3) × no_ammo(0.2)
Cover score   = under_fire(0.4) × cover_nearby(0.4) × health_low(0.2)
→ Highest score wins
```
