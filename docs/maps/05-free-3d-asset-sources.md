## 6. Tile Mesh Templates (for 8-4-4 Sub-Tile System)

These need to be simple geometric shapes — the 4 sub-tile quadrant meshes (Fill, Edge, OuterCorner, InnerCorner). Options:

| Approach | Source | Notes |
|----------|--------|-------|
| **meshy.ai** (decided) | https://www.meshy.ai/ | Generate the 4 shapes per surface type with consistent style. Prompt: "quarter tile terrain piece, top-down view, [surface type], game asset" |
| **Procedural fallback** | Code-generated in Stride | Simple geometric primitive: flat quad with beveled edges. Use as placeholder while meshy.ai assets are produced. |
| **Blender + export** | https://www.blender.org/ | Model the 4 shapes once in Blender, export as glTF. Apply per-surface-type materials at runtime via triplanar shader. |

## 9. Import Notes for Stride

- **Preferred format:** glTF 2.0 (`.glb` / `.gltf`) — best Stride support
- **Also supported:** FBX, OBJ (with limitations)
- **Texture format:** PNG or JPEG for albedo/normal/roughness maps
- **Scale:** Ensure models are in meters (1 unit = 1m) to match tile size
- **Poly budget per prop:** < 5K triangles (Low), < 15K (Medium), < 50K (High quality preset)
- **Poly budget per building:** < 20K (Low), < 50K (Medium), < 100K (High)
